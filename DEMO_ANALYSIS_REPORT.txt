4. STOCK REDUCTION LOGIC
===============================================================================

Status: ✅ FULLY IMPLEMENTED

  Backend: server/src/services/inventoryService.js (updateInventory)
  Database: server/src/models/Inventory.js
  Database: server/src/models/StockLedger.js

Process Flow With Implementation Details:

Step 1: Stock Validation
  Location: salesController.js lines 41-52
  - Loops through all sale items
  - Checks current inventory for each product
  - Compares available qty against requested qty
  - Returns error if qty insufficient

Step 2: Sale Record Creation
  Location: salesController.js lines 61-75
  - Generates unique bill number via generateBillNo()
  - Creates Sale document with:
    * billNo (unique identifier)
    * shopId (from user token)
    * items array (product, quantity, price, subtotal)
    * totalAmount (calculated)
    * paymentMethod (cash/card/online/etc)
    * paymentDetails (full payment info)
    * paymentStatus (completed)
    * saleDate (timestamp)

Step 3: Inventory Update
  Location: For each sale item:
  - Calls inventoryService.updateInventory()
  - Parameters: shopId, productId, -quantity (negative to decrease)
  - Transaction type: 'sale_out'
  - Reference: sale._id

Step 4: Ledger Entry Creation
  Location: inventoryService.js
  - Creates StockLedger entry with:
    * shopId
    * productId
    * transactionType: 'sale_out'
    * quantity: -N (negative)
    * referenceId: sale._id
    * referenceType: 'sale'
    * transactionDate: now

Step 5: Inventory Quantity Update
  Location: Inventory.js document
  - Uses MongoDB $inc operator
  - Increments quantity by -N (reduces stock)
  - Updates lastUpdated timestamp

Verification Points:
  ✅ Inventory.findOne() before sale - shows qty before
  ✅ After sale - qty decreased by exact sale quantity
  ✅ StockLedger entries created for each item
  ✅ Multiple sales on same product tracked properly
  ✅ Audit trail maintained in StockLedger
